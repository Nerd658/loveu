## 2024-05-24 - Canvas Interactive Element Accessibility
**Learning:** When using `<canvas>` elements as interactive buttons, they lack native semantic states. Without explicit visual hover/focus states and programmatic ARIA updates (like `aria-disabled` and `aria-label`), screen readers and keyboard users cannot perceive the interaction state or when a state change occurs (e.g., after an animation).
**Action:** Always provide explicit visual hover/focus states via CSS for interactive `<canvas>` elements and manually update `aria-disabled` and `aria-label` attributes to communicate their interactive states and state changes to screen readers.
